# 删除所有的语音包

在 Windows 10 中（尤其是使用 Tiny10 等精简系统，或者通过离线命令强刷过中文包后），系统里可能会残留一些用不上的英文、日文等语音包组件（Speech Packages）。这些语音包不仅占用空间，有时还会导致后台服务（如 Cortana 或 Windows Speech）暗中消耗资源。

要**彻底、干净地删除所有不需要的语音包**，使用传统的“设置”界面往往删不干净，最强硬且彻底的方法是通过 **PowerShell 命令行**或 **DISM 部署工具**进行底层物理擦除。

以下为你提供两种卸载方案：

## 方案一：使用 PowerShell 命令自动一键清理（最推荐 ⭐⭐⭐⭐⭐）

这个方法利用脚本直接获取当前系统安装的所有语音包，并强制将其卸载，非常省时省力。

### 操作步骤：

1. 右键点击 **开始按钮**，选择 **“Windows PowerShell (管理员)”** 或 **“终端 (管理员)”**。
2. 复制下方命令，在 PowerShell 窗口中点击右键粘贴，然后按 **回车**：

   ```PowerShell
   Get-WindowsPackage -Online | Where-Object {$_.PackageName -like "*Speech*"} | ForEach-Object { Remove-WindowsPackage -Online -PackageName $_.PackageName -NoRestart }
   ```

1. **命令原理解析：**
   - 该命令会自动扫描系统底层所有带有 `Speech`（语音）关键字的语言组件。
   - 自动对它们执行 `Remove`（强制卸载）操作。
2. 运行过程中屏幕会闪过进度条，等待命令完全执行完毕（重新出现光标输入提示）后，**重启电脑**即可。

## 方案二：使用 DISM 精准手动定向爆破（针对顽固残留）

如果方案一运行后发现某些语音包依然存在，说明它们属于深度残留的系统程序包（Capabilities）。我们可以用系统自带的 DISM 工具进行精准定点清除。

### 操作步骤：

1. 同样以**管理员身份**打开 **PowerShell** 或 **命令提示符 (CMD)**。

2. 输入以下命令，查看当前系统到底挂载了哪些语音包：

   `DOS`

   ```
   dism /online /get-capabilities | findstr "Language.Speech"
   ```

3. 此时屏幕上会列出类似下面的具体语音包名称：

   - `Language.Speech~~~en-US~0.0.1.0` (美式英语语音包)
   - `Language.Speech~~~ja-JP~0.0.1.0` (日语语音包)
   - `Language.Speech~~~zh-CN~0.0.1.0` (简体中文语音包)

4. **执行卸载：** 根据你需要删除的语种，复制对应的名字，运行卸载命令。例如，**如果你想把英文语音包彻底干掉**，运行：

   DOS

   ```
   dism /online /remove-capability /capabilityname:Language.Speech~~~en-US~0.0.1.0
   ```

   *(如果你想删日语，就把最后的 `en-US` 换成 `ja-JP`。注意：**千万不要删掉 `zh-CN`**，否则你的中文系统可能会出现某些自带应用闪退或小娜组件报错。)*

5. 提示“操作成功完成”后，重启计算机即可。

## 💡 补充：关闭语音服务，防止系统偷偷下载回来

Windows 10 在联网状态下，有时会自动激活“在线语音识别”并偷偷把语言包重新下载回来。为了永绝后患，建议顺手关闭这个开关：

1. 打开 **设置 (Settings) -> 隐私 (Privacy) -> 语音 (Speech)**。
2. 将 **“联机语音识别” (Online speech recognition)** 的开关彻底**关闭 (Off)**。

这样操作后，系统底层就不会再有任何冗余的语音引擎在后台占用你的 C 盘空间了。
