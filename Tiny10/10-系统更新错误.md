## 系统更新错误

Tiny10 系统在更新时遇到 **0x80073701** 错误，这在精简版系统里其实挺常见的。这个错误代码在微软官方的定义是 `ERROR_SXS_ASSEMBLY_MISSING`，通俗来说就是：**更新程序在安装时，发现系统里缺少了它所依赖的某些基础组件或语言包。**

因为 Tiny10 为了追求极度精简，作者精简掉了大量的系统组件（如部分语言包、Manifest 清单文件、WinSxS 缓存等）。当微软发布的累积更新（Cumulative Update）试图去寻找这些被删掉的底层文件时，找不到就会直接卡死并报错。

要解决这个问题，可以按照以下由浅入深的步骤来尝试修复：

### 第一步：清理组件存储（最有效的常规手段）

有时候 WinSxS 组件库里的残留缓存或错误的挂起任务会引发冲突，可以通过部署映像服务和管理工具（DISM）来进行深度清理。

以**管理员身份**打开命令提示符（CMD），依次复制并运行以下命令：

DOS

```
dism /online /cleanup-image /startcomponentcleanup
```

> **注意**：这个命令会清理掉旧的组件版本，执行时间可能较长（几分钟到十几分钟不等），请耐心等待它跑到 100%。

完成后，紧接着运行系统修复：

DOS

```
sfc /scannow
```

运行完后**重启电脑**，再次尝试检查更新。

### 第二步：重置 Windows 更新组件缓存

如果第一步无效，可能是 Tiny10 的更新缓存目录出现了逻辑混乱。我们可以强制停止更新服务，把缓存文件夹改名重置。

以**管理员身份**打开 CMD，逐行输入并回车：

DOS

```
net stop wuauserv
net stop cryptSvc
net stop bits
net stop msiserver
ren C:\Windows\SoftwareDistribution SoftwareDistribution.old
ren C:\Windows\System32\catroot2 catroot2.old
net start wuauserv
net start cryptSvc
net start bits
net start msiserver
```

这些命令会停掉更新相关的服务，把 `SoftwareDistribution` 和 `catroot2` 这两个存放更新缓存的文件夹重命名（相当于格式化了更新缓存），然后重启服务。完成后再次尝试更新。

### 第三步：去微软官网手动下载补丁包

如果系统自带的自动更新总是报错，可以绕过它，直接去官方“开小灶”。

1. 打开 Windows 设置中的更新页面，记下那个**安装失败的补丁号**（例如 `KB50XXXXX`）。
2. 用浏览器打开 [微软更新产品目录网站 (Microsoft Update Catalog)](https://www.catalog.update.microsoft.com/)。
3. 在右上角搜索框输入该补丁号（如 `KB50XXXXX`）。
4. 根据你的 Tiny10 版本（通常是 x64 或者是 x86），下载对应的独立安装包（`.msu` 格式）。
5. 断开网络，双击运行下载好的安装包进行手动安装。

### 💡 终极提醒：精简版系统的两难境地

如果上述方法全部试过依然报错 0x80073701，那往往意味着当前尝试安装的这个微软补丁，**强依赖于某个已经被 Tiny10 彻底物理删除、且无法通过常规手段恢复的底层组件。**

对于这个问题，有经验的折腾党通常有两种选择：

1. **直接封印更新**：使用工具（如 O&O ShutUp10 或 StopUpdates10）彻底关闭 Tiny10 的系统更新。毕竟精简版系统追求的是轻量和稳定，不折腾频繁的累积更新也是一种用法。
2. **更换系统底座**：如果对安全性或新功能有刚需，建议更换为官方原版的 **Windows 10 LTSC (长期服务版)** 或 **Windows 10 IoT Enterprise LTSC**。官方 LTSC 同样没有小娜、应用商店等冗余垃圾，占用资源极低，但它的组件是完整的，可以完美通过每月的安全更新，省去了精简版系统经常断更或报错的烦恼。
