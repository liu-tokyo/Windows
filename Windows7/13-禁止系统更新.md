## 禁止系统更新

Windows 7 既然已经停止支持，关闭其更新服务确实是节省内存和 CPU 资源的明智做法，因为其后台服务（如 `svchost.exe` 下的 `wuauserv`）有时会因搜寻不存在的更新而产生极高的资源占用（即著名的“内存泄漏”或“CPU 100%”问题）。

以下是**最彻底**的关闭方案：

### 1. 修改系统服务（直接切断运行）

这是最立竿见影的方法，可以防止 Windows Update 进程在后台常驻内存。

1. 按下键盘上的 `Win + R` 键，输入 `services.msc` 并回车。
2. 在弹出的服务列表中，拉到最底部找到 **Windows Update**。
3. 双击它，在“常规”选项卡中：
   - 点击 **停止** 按钮。
   - 将 **启动类型** 修改为 **禁用**。
4. 点击顶部的“恢复”选项卡，将“第一次失败”、“第二次失败”等全部设置为 **无操作**（防止系统自动重启该服务）。
5. 点击“确定”。

### 2. 控制面板设置（更改逻辑）

防止系统界面反复弹出更新提醒。

1. 打开 **控制面板** > **系统和安全** > **Windows Update**。
2. 点击左侧的 **更改设置**。
3. 在“重要更新”下拉菜单中，选择 **“从不检查更新（不推荐）”**。
4. 取消勾选下方的所有复选框（包括“允许所有用户...”等），点击确定。

### 3. 修改组策略（双重保险）

如果你使用的是专业版或旗舰版 Windows 7，这可以从底层锁死更新。

1. 按下 `Win + R`，输入 `gpedit.msc` 并回车。
2. 导航至：`计算机配置` > `管理模板` > `Windows 组件` > `Windows Update`。
3. 在右侧找到 **配置自动更新**，双击它。
4. 选择 **已禁用**，点击确定。

------

### 💡 进阶：如何进一步释放内存？

关闭更新后，建议顺便进行以下操作，彻底清理因更新产生的缓存垃圾：

- **清理更新缓存：**
  1. 按上文方法停止 Windows Update 服务。
  2. 打开 `C:\Windows\SoftwareDistribution` 文件夹。
  3. **删除** 该文件夹内的所有内容（这些是已下载的更新安装包和缓存日志，通常有几 GB）。
  4. 重新启动电脑。
- **清理启动项：** 按下 `Win + R` 输入 `msconfig`，在“启动”选项卡中禁用不需要的软件自动启动。

**注意：** 关闭更新后，系统将不再获得安全补丁。虽然 2026 年的 Win7 已经没有官方补丁了，但请务必配合一款轻量级的杀毒软件（如火绒或类似的轻量工具）来保证安全。
