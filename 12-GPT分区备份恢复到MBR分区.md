# GPT分区备份恢复到MBR分区

 DiskGenius 备份的分区文件（通常是 `.pmf` 格式）记录的是**分区内部的文件系统和数据**，而 MBR 或 GUID（GPT）是**整个硬盘的分区表格式**。这两者是相互独立的。

也就是说，无论你当时备份这个分区时，原硬盘是 GPT 还是 MBR，你都可以把它恢复到一块 MBR 格式的硬盘上。

不过，在恢复过程中，根据你的**具体用途（是系统盘还是纯数据盘）**，有几个关键的隐藏坑需要特别注意：

## 情况一：如果你恢复的是“纯数据盘”

如果这个分区只用来放电影、文档或游戏，操作非常简单，直接恢复即可，没有任何格式冲突。

1. 打开 DiskGenius，选中 MBR 硬盘上的目标分区。
2. 点击顶部菜单的 **“工具” -> “从镜像文件还原分区”**。
3. 选择你的备份文件，点击开始即可。

## 情况二：如果你恢复的是“C 盘（系统盘）”

这是最容易翻车的地方。如果原本的系统是在 GPT（UEFI）模式下备份的，现在想恢复到 MBR（Legacy）硬盘上当系统盘启动，**直接还原会导致系统无法开机**。

如果你要恢复的是系统盘，请务必按照以下步骤处理：

### 1. 确认 MBR 硬盘的“激活”状态

MBR 格式的硬盘必须有一个“活动分区”（Active），系统才能引导。

- 还原分区后，在 DiskGenius 中右键点击这个恢复好的 C 盘。
- 检查 **“激活当前分区”** 选项是否可选。如果可选，请点击它，将其设置为活动分区（分区名字前会多一个红色的小旗子）。

### 2. 修复引导（最关键的一步）

GPT 用的引导文件在 ESP 分区（EFI 模式），而 MBR 用的引导文件在 C 盘根目录（NTFS 传统模式）。直接恢复过去的系统缺少 MBR 引导代码。

- 还原分区后，**不要立刻重启电脑**。

- 在 DiskGenius 中选中该 MBR 硬盘，点击顶部菜单的 **“磁盘” -> “重建主引导记录 (MBR)”**。

- 接着，建议使用 NTBOOTAUTOFIX 等引导修复工具，或者在 PE 系统中打开命令行，运行以下命令来修复 C 盘的传统引导：

  DOS

  ```
  bcdboot C:\Windows /s C: /f BIOS
  ```

  *(注意：请根据实际情况将 `C:` 替换为你恢复后的系统盘盘符)*

### 3. 修改主板 BIOS 设置

由于你换成了 MBR 分区，重启电脑进入主板 BIOS 时：

- 必须将 **Boot Mode（启动模式）** 从 `UEFI` 修改为 `Legacy`（或者开启 `CSM` 兼容模式）。
- 否则主板依然会用 UEFI 模式去寻找 GPT 分区，导致找不到启动盘。

## 总结

DiskGenius 的分区备份**不挑分区表格式**，随便跨格式恢复。但如果是**系统盘跨格式恢复**，一定要记得“激活分区 + 重建 MBR + 修复 BCD 引导 + 核心 BIOS 改 Legacy” 这四部曲。
