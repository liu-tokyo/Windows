## 如何开启快速启动



这可能是因为 Windows 10 的**快速启动（Fast Startup）**机制被某些设置、硬件环境或系统错误给“绕过”了。

首先你要明确一个概念：**“快速启动”本质上是“半个休眠”**。当你点击关机时，它会把内核和驱动状态存入 `hiberfil.sys`；当你再次开机时，直接读取这个文件。

以下是快速启动失效的几个常见原因及排查方案：

------

### 1. 核心前提：休眠文件是否被关闭？

这是最常见的原因。如果你之前为了给 C 盘瘦身运行了 `powercfg -h off`，那么**快速启动会由于依赖的休眠功能消失而自动失效**，哪怕设置里勾选了也没用。

- **检查方法：** 看 C 盘根目录（需开启隐藏文件可见）是否有 `hiberfil.sys`。
- **修复方法：** 管理员模式运行 CMD，输入： `powercfg -h on` 然后重启电脑。

### 2. 你是不是点了“重启”而不是“关机”？

这是一个设计细节：

- **点击“关机”：** 触发快速启动。
- **点击“重启”：** Windows 会完整关闭所有进程并重新加载系统，**不触发快速启动**。

> 如果你是通过“重启”来测试速度的，那你永远感受不到快速启动。

### 3. BIOS/UEFI 里的设置冲突

快速启动需要主板固件的配合。

- 检查 BIOS 设置中的 **Fast Boot** 或 **Ultra Fast Boot** 是否开启。
- 确保你的磁盘分区表是 **GPT** 格式，且启动模式是 **UEFI**。如果是过时的 Legacy 模式，快速启动的效果会大打折扣。

### 4. 驱动程序不兼容（最隐蔽的原因）

有些旧驱动（特别是旧版的 Intel 管理引擎接口 MEI 驱动）不支持快速启动的电源状态，导致关机时无法正常写入休眠文件，系统只好被迫进行“彻底关机”。

- **排查方法：** 1.  按下 `Win + X` -> **事件查看器**。 2.  导航至：**Windows 日志 -> 系统**。 3.  右侧点击“筛选当前日志”，在“事件 ID”处输入 `27` 或 `104`。 4.  如果看到大量关于电源切换失败的警告，说明某个驱动在阻止关机时的状态保存。

### 5. 开启“真正的”快速启动（命令强制开启）

如果设置界面里压根找不到这个选项：

1. 管理员运行 CMD，输入： `powercfg /hibernate-timeout 0`
2. 进入 **控制面板 -> 硬件和声音 -> 电源选项 -> 选择电源按钮的功能**。
3. 点击“更改当前不可用的设置”，勾选“启用快速启动”。

------

### ⚡ 进阶技巧：如何判断它生效了？

想知道你的快速启动到底有没有发挥作用，可以这样看：

1. 开机进入桌面，按 `Ctrl + Shift + Esc` 打开 **任务管理器**。
2. 点击 **性能 -> CPU**。
3. 查看右下角的 **“正常运行时间”**。

> **结论：** 如果你刚刚“关机再开机”，但这里的运行时间显示是 **好几天** 甚至更久，说明快速启动**生效了**（它没有清空计时器）。如果时间只有几秒钟，说明是**完整关机**，快速启动没起作用。



## 排查快速启动无效的原因

### 1. 检查控制面板中的“大开关”

即便命令行开启了休眠，UI 界面里的“快速启动”勾选框可能并没有被勾选，或者被系统策略锁死。

1. 打开 **控制面板** -> **硬件和声音** -> **电源选项**。
2. 点击左侧的 **“选择电源按钮的功能”**。
3. 点击顶部的 **“更改当前不可用的设置”**（需要管理员权限）。
4. 检查底部的 **“启用快速启动 (推荐)”** 是否被勾选。如果没勾选，请勾选并保存修改。

------

### 2. 排除“重启”与“关机”的区别

这是一个极其常见的小误区：

- **点击“重启”：** Windows 会执行完整的冷启动，**不应用**快速启动。这是为了确保系统在遇到问题时能彻底刷新。
- **点击“关机”：** 才会应用快速启动。

> **测试方法：** 请确保你是点击“关机”后再按电源键开机，而不是点击“重启”。

------

### 3. 驱动程序拦截了“混合关机”

快速启动的本质是“注销用户会话 + 将系统内核休眠”。如果有驱动程序不支持电源状态转换，Windows 会在尝试进入快速启动失败后，自动回退到传统的彻底关机。

**主要嫌疑人：**

- **显卡驱动：** 建议去官网下载最新版驱动，不要使用 Windows Update 自带的。
- **Intel 管理引擎 (IME) 驱动：** 这个驱动直接控制电源状态。如果版本过旧或损坏，快速启动会失效。

------

### 4. 检查 BIOS/UEFI 设置

如果主板的底层设置关闭了相关电源管理，Windows 层的设置也会失效。

- 进入 BIOS，确保开启了 **ACPI 相关电源管理**。
- 检查是否有 **Fast Boot** 选项，虽然它和 Windows 的快速启动不是一个东西，但建议也开启。

------

### 5. 查看系统日志确认原因

我们可以通过 Windows 事件查看器找到“为什么上次关机没能走快速启动”的证据：

1. 右键点击“开始”，选择 **事件查看器**。
2. 导航到：**Windows 日志 -> 系统**。
3. 点击右侧的 **“筛选当前日志”**，在事件 ID 中输入 `27`（代表内核启动/快速启动状态）。
4. 查看日志描述：
   - 如果看到 *“该启动类型为 0x1（快速启动）”*，说明有效。
   - 如果看到 *“该启动类型为 0x0（冷启动）”*，它通常会附带原因（例如某个驱动程序报错导致失败）。

------

### 6. 一个奇招：重置电源方案

如果上述都对，可能是电源配置方案损坏了： 在管理员 CMD 中输入： `powercfg -restoredefaultschemes` 这会重置所有电源设置。执行后，记得回到控制面板重新检查“快速启动”勾选框是否勾选。
