# 如何减少日志输出

在 Windows 11 + 机械硬盘（HDD）的极限生存挑战中，你抓到了一个非常硬核且关键的点：**系统日志和事件跟踪**。

Windows 11 默认是个“话痨”，系统底层的 ETW（Event Tracing for Windows，Windows 事件跟踪）机制、组件服务、遥测和安全模块在后台一刻不停地往硬盘里写入各种 `.evtx` 日志文件和 `.etl` 跟踪文件。

机械硬盘的磁头本来就慢，一旦遇到这种**高频、细碎的随机写入（4K随机写）**，磁头就会卡在日志扇区疯狂寻道，直接导致你在前台操作时发生“卡顿”或“100% 占用”。

通过以下四步深层优化，可以把 Windows 11 的日志读写降到最低，彻底解放机械硬盘的磁头：

## 一、 禁用高频“吃硬盘”的事件跟踪日志（关键步骤）

Windows 底层有几个专门用来收集性能和网络数据的实时跟踪器，它们是导致 HDD 频繁闪烁的隐形杀手。

### 1. 关闭 WPR 性能跟踪

- **操作：**

  1. 在开始菜单搜索 `cmd`，右键选择 **“以管理员身份运行”**。

  2. 输入以下命令并回车，彻底关闭系统自带的性能记录器：

     DOS

     ```
     wpr -cancel
     ```

### 2. 在计算机管理中停用高频跟踪器

- **操作：**
  1. 按 `Win + X` 键，选择 **“计算机管理”**。
  2. 依次展开：**系统工具 -> 性能 -> 数据收集集 -> 启动事件跟踪会话**。
  3. 在右侧列表中，你会看到很多跟踪项。找到以下几项（它们最喜欢高频写盘），分别右键点击它们，选择 **“属性”**，在“会话”选项卡中**取消勾选“启用”**，点击确定：
     - `EventLog-AirSpace`（图形界面相关日志）
     - `WdiContextLog`（无线诊断相关日志）
     - `WiFiSession`（Wi-Fi连接日志）
     - `CloudFilesDiagnosticLog`（OneDrive等云文件同步诊断）
  4. *注意：不要动带有 `NT Kernel` 或 `Circular` 字样的核心项，其余带有 `Diagnostic`（诊断）或 `Telemetry`（遥测）的都可以放心禁用。*

## 二、 调整事件查看器（Event Viewer）的写入策略

既然无法完全彻底消灭所有系统日志，我们可以通过修改策略，限制日志文件的体积，并且**禁止它把日志写在机械硬盘上**（如果你有内存盘或小固态）。

### 1. 限制日志文件最大大小

默认情况下，每个分类的日志（如应用程序、系统、安全）可以长到 20MB。我们可以把它限制得极小，写满后直接覆盖，不让它产生大文件。

- **操作：**
  1. 按 `Win + R` 输入 `eventvwr.msc` 打开事件查看器。
  2. 展开 **Windows 日志**，分别右键点击 **应用程序 (Application)**、**安全 (Security)**、**设置 (Setup)**、**系统 (System)**，选择 **“属性”**。
  3. 将“最大日志大小(KB)”手动修改为 **`1024`** (即 1MB)。
  4. 确保因日志满时的处理方式勾选为 **“按需覆盖事件(旧事件优先)”**。

### 2. 进阶折腾：把日志目录迁移到 RAMDisk（内存盘）

如果你参考了上一篇方案，利用多余内存划分了 RAMDisk（内存盘，假设盘符为 `Z:`），你可以强行把 Windows 日志的生成路径改到内存里。**既然是在内存里读写，机械硬盘的损耗直接归零！**

- **操作：** 在上述事件查看器的日志“属性”窗口中，有一个 **“日志路径”**。你可以把默认的 `C:\Windows\System32\Winevt\Logs\...` 修改为 `Z:\Logs\...`（需要在内存盘先建好 Logs 文件夹）。

## 三、 彻底砍断“遥测（Telemetry）”与错误报告

微软的遥测系统会收集你的使用习惯、软件崩溃崩溃信息并生成庞大的日志文件打包上传，这对机械硬盘是极大的负担。

### 1. 禁用 Windows 错误报告服务

- **操作：** 按 `Win + R` 输入 `services.msc` 打开服务。
- 找到 **Windows Error Reporting Service (Windows 错误报告服务)**，右键属性，将启动类型改为 **“禁用”** 并停止服务。

### 2. 彻底关闭客户体验改善计划（注册表修剪）

- **操作：**
  1. 按 `Win + R` 输入 `regedit` 打开注册表编辑器。
  2. 定位到路径：`HKEY_LOCAL_MACHINE\SOFTWARE\Policies\Microsoft\Windows\DataCollection`
  3. 在右侧空白处右键 -> **新建 -> DWORD (32 位)值**，命名为 **`AllowTelemetry`**。
  4. 双击它，确保它的数值数据为 **`0`**（0 代表完全关闭遥测）。

## 四、 组策略限制 CBS 组件组件日志

Windows 在更新或安装组件时，会在 `C:\Windows\Logs\CBS` 目录下生成极为庞大的 `CBS.log`。有时候这个日志会因为 Bug 疯狂自我复制，塞满几十个 GB，把机械硬盘活活卡死。

- **操作（关闭 Windows Modules Installer 的过度诊断）：**
  1. 按 `Win + R` 输入 `gpedit.msc` 打开组策略编辑器（如果系统是家庭版，需先执行脚本开启组策略）。
  2. 计算机配置 -> 管理模板 -> 系统 -> 故障转移聚类（或者直接搜索“组件相关诊断”）。
  3. 更简单的做法是：定期去 `C:\Windows\Logs\CBS` 检查，如果发现有多个以 `winsta` 或 `cbs` 开头的巨型 `.log` 或 `.cab` 压缩包，直接果断删除它们。
